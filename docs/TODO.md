# TODO: Libarally 将来的な拡張課題

## 1. 優先度の高い拡張機能
### 1.1. 通知システム (Notification System)
- **概要**: 予約した本の準備が整った際や、返却期限が近づいた際にユーザーへ通知を届ける仕組み。
- **背景**: 現時点ではコア機能の完成を優先し、本機能の実装は将来課題として保留とする。
- **実装内容**:
    - 予約引き当て時の自動通知生成。
    - ヘッダーへの未読カウント表示。
    - 通知一覧画面の構築。

### 1.2. 外部API連携による書籍情報の自動取得
- **概要**: ISBNを入力またはスキャンするだけで、タイトル、著者、出版社、書影、あらすじを自動取得する機能。
- **対象API**: 楽天ブックスAPI、Google Books API等。
- **メリット**: 管理者の登録コストを劇的に削減し、データの正確性を向上させる。

### 1.3. 予約ロジックの高度化（在庫引き当て）
- **概要**: 在庫がある状態で予約が行われた際、その場で特定の「本（個体）」を確保（RESERVED状態へ変更）するロジック。
- **現状**: 現在は「誰かが借りている場合のみ予約可能」という暫定的なUIガードで対応中。

## 2. コミュニティ・UX拡張
### 2.1. レビュー・評価機能
- **内容**: 読んだ本に対して 5 段階評価とコメントを投稿できる機能。
- **目的**: 社内での知見共有をさらに活性化させる。

### 2.2. レコメンドエンジン
- **内容**: 貸出履歴に基づき、「この本を読んだ人は、こんな本も読んでいます」を表示する機能。

### 2.3. カート（まとめて貸出）機能
- **内容**: 複数の本を一時的にストックし、一括で貸出・予約手続きを行う機能。
